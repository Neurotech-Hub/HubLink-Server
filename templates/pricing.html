How it Works

Nodes

Hublink requires that you have ESP32-based nodes in order to communicate over Bluetooth Low Energy (BLE) to gateways. This is made simple via the open-source Hublink Node Library for the Arduino IDE.

Gateways

Hublink gateways are pre-configured Raspberry Pi devices that simply need a power outlet and internet connection. We offer these nearly at cost! Your gateways will sync settings using our hublink.cloud Dashboard to enforce consistency across your deployment.

Not using the cloud?

Gateways can run in stand-alone mode and still collect node data. However, conflict resolution between gateways is done at the cloud-level, so you will need to develop (or hire us) a custom workflow.

Hublink Cloud

The Hublink Cloud web-app is the primary link between nodes, gateways, and cloud storage. We currently rely on Amazon S3 for long-term, persistent, reliable storage. After your purchase of a Hublink gateway, we will provision your Amazon S3 bucket and you will own those data. Amazon S3 provides a rich API itself, which can be augemented with our open-source Hublink Client.
