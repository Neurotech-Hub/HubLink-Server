Hublink is a turn-key system for gathering, managing, and viewing data from multiple node and gateway devices.

Nodes are any "edge" device that collect data, typically onto an SD card. Gateways are centralized devices (one or many) used for gathering data from nodes (one or many). Gateways use hublink.cloud endpoints to coordinate settings and data transfer to the cloud (Amazon S3).

Core Components

The Hublink architecture is composed of:

  1. Hublink Node: An embedded, Arduino-based device (e.g., ESP32-S3) using the hublink library to mediate efficient file transfer over Bluetooth Low Energy.
  2. Hublink Gateway: Linux-based (e.g., Raspberry Pi) deployable hardware to find, connect, and manage data transfer from nodes and sync with cloud storage.
  3. Hublink.cloud: A web-based app to sync gateway settings and manage all aspects of the Hublink system.
  4. Hublink Client: A client library written in Python to interact, consume, and sync with Hublink-created Amazon S3 data.
