Hublink is a turn-key system for gathering, managing, and viewing data from multiple node and gateway devices.

Nodes are any "edge" device that collect data, typically onto an SD card. Gateways are centralized devices (one or many) used for gathering data from nodes (one or many). Gateways use hublink.cloud API endpoints to coordinate settings and data transfer to the cloud (Amazon S3).

Core Components

Component Description Technology
Hublink Node
Open Source
 An embedded device using the Hublink library to mediate efficient file transfer. Arduino IDE, ESP32, Bluetooth Low Energy
Hublink Gateway
Proprietary
A device used to find, connect, and manage data transfer from nodes to the cloud. Linux, Raspberry Pi
Hublink.cloud
Proprietary
A web-app with API endpoints to sync gateway settings. Managed Server
Hublink Client
Open Source
 A client library to interact, consume, and sync with Hublink-created data. Amazon S3, Python

Features

Settings Config

Easily set credentials, file size limits, and other configurations to manage cloud storage.

Data Sync

Automatically back up experiment data to centralized cloud storage for easy access.

Dashboard

Monitor devices, receive alerts, and track experiment progress in real-time on an intuitive dashboard.

REST API

Programmatically check file existence and output account settings in JSON format for integration.

Ultra Shareable

Share your dashboard anywhere, anytime once you have your <secret_url>.

Your dashboard will appear at: https://hublink.cloud/<secret_url>

Mobile Native

Full access on-the-go with the mobile-friendly web app. Manage and monitor from your smartphone or tablet.
